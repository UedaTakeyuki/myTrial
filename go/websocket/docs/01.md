# Cloudflare Workers (Durable Objects) & Go WebSocket 接続・仕様まとめ

## 1. 動作確認済みの完全なコード一式

### 🏢 Cloudflare Workers / Durable Object 側 (src/index.js)
```javascript
import { DurableObject } from "cloudflare:workers";

export class MyDurableObject extends DurableObject {
	constructor(ctx, env) {
		super(ctx, env);
	}

	async fetch(request) {
		const pair = new WebSocketPair();
		const clientWebSocket = pair[0];
		const serverWebSocket = pair[1];

		// DO側でWebSocketの管理を引き受ける
		this.ctx.acceptWebSocket(serverWebSocket);

		// 101 Switching Protocols レスポンスを返して接続を確立
		return new Response(null, {
			status: 101,
			webSocket: clientWebSocket,
		});
	}

	// 💡 ポイント: 引数の message は ArrayBuffer または string のため型判定が必要
	async webSocketMessage(webSocket, message) {
		let receivedText = "";

		if (message instanceof ArrayBuffer) {
			receivedText = new TextDecoder().decode(message);
		} else {
			receivedText = typeof message === "string" ? message : JSON.stringify(message);
		}
		
		// 文字列を反転する（例: "hello" -> "olleh"）
		const reversedText = receivedText.split("").reverse().join("");

		// クライアントへ直接返信
		webSocket.send(`[DOからの返事]: ${reversedText}`);
	}

	async webSocketClose(webSocket, code, reason, wasClean) {
		console.log("WebSocket 接続が閉じられました");
	}
}

export default {
	async fetch(request, env, ctx) {
		// WebSocketの接続要求（Upgradeヘッダー）があればDOへ転送
		if (request.headers.get("Upgrade") !== "websocket") {
			return new Response(getHTMLPage(), {
				headers: { "Content-Type": "text/html;charset=UTF-8" }
			});
		}

		const stub = env.MY_DURABLE_OBJECT.getByName("foo");
		return await stub.fetch(request);
	},
};

function getHTMLPage() {
	return `
	<!DOCTYPE html>
	<html>
	<head><title>DO WebSocket Test</title></head>
	<body style="font-family: sans-serif; padding: 20px;">
		<h2>Durable Object WebSocket デモ</h2>
		<input type="text" id="msgInput" value="hello" style="padding:5px; width:200px;">
		<button onclick="sendMessage()">送信</button>
		<div id="output" style="margin-top:20px; padding:10px; background:#f0f0f0; min-height:100px; white-space:pre-line;"></div>

		<script>
			// 安全にプロトコルとホスト名を置換して接続URLを作成
			const wsUrl = (window.location.protocol === "https:" ? "wss://" : "ws://") + window.location.host + "/";
			const ws = new WebSocket(wsUrl);

			ws.onmessage = (event) => {
				document.getElementById("output").innerText += event.data + "\\n";
			};

			ws.onopen = () => {
				document.getElementById("output").innerText += "[システム]: 接続完了しました！\\n";
			};

			ws.onerror = () => {
				document.getElementById("output").innerText += "[エラー]: 接続でエラーが発生しました\\n";
			};

			function sendMessage() {
				const input = document.getElementById("msgInput");
				ws.send(input.value);
			}
		</script>
	</body>
	</html>
	`;
}
```

### 🐹 Go言語 クライアント側 (main.go)
```go
package main

import (
	"crypto/tls"
	"log"
	"os"
	"os/signal"
	"time"

	"://github.com"
)

func main() {
	interrupt := make(chan os.Signal, 1)
	signal.Notify(interrupt, os.Interrupt)

	// 💡 ポイント: wrangler dev で起動している正しいIPアドレスとプロトコル(ws://)を指定
	url := "ws://192.168.11.12:8787"
	log.Printf("接続中: %s", url)

	// 自作証明書やローカルHTTPS(wss)での検証エラーを回避する設定を入れたカスタムDialer
	dialer := websocket.Dialer{
		TLSClientConfig: &tls.Config{InsecureSkipVerify: true},
	}

	c, _, err := dialer.Dial(url, nil)
	if err != nil {
		log.Fatal("ダイアルエラー:", err)
	}
	defer c.Close()

	done := make(chan struct{})
	go func() {
		defer close(done)
		for {
			_, message, err := c.ReadMessage()
			if err != nil {
				log.Println("読み取りエラー:", err)
				return
			}
			log.Printf("受信: %s", message)
		}
	}()

	ticker := time.NewTicker(time.Second)
	defer ticker.Stop()

	for {
		select {
		case <-done:
			return
		case t := <-ticker.C:
			msg := "こんにちは！ 現在時刻: " + t.Format("15:04:05")
			err := c.WriteMessage(websocket.TextMessage, []byte(msg))
			if err != nil {
				log.Println("書き込みエラー:", err)
				return
			}
		case <-interrupt:
			log.Println("割り込みを検知しました。切断します...")

			err := c.WriteMessage(websocket.CloseMessage, websocket.FormatCloseMessage(websocket.CloseNormalClosure, ""))
			if err != nil {
				log.Println("クローズメッセージ書き込みエラー:", err)
				return
			}
			
			select {
			case <-done:
			case <-time.After(time.Second):
			}
			return
		}
	}
}
```

## 2. Durable Objects (DO) の「10ms制限」の正しい仕組み

Durable ObjectsのStandardプランにおける「10ms制限」は、**「1本のWebSocketセッションの維持時間」ではありません。**

### ⏱️ 制限の単位
* **「1回のイベント（関数）に入ってから、処理を抜けるまで」** の実質的なCPU消費時間が対象。
* 例：Goからメッセージが1個届き、`webSocketMessage()` が起動して処理を終えるまでが「1回」。次の秒に届いたメッセージは、また独立した「新しい1回」としてカウントされる。
* WorkersからDOのメソッド（RPC機能や `fetch`）を呼び出した場合も同様に、**「呼び出されたDO側の関数に入ってから `return` で抜けるまで」** が1回の単位。

### ⏳ カウントされない時間（タイマー一時停止）
* 処理の途中で `await fetch()` を使って外部APIを叩いたり、`await this.ctx.storage.get()` でデータベースの読み書きを待っている時間は、**CPUタイマーが一時停止**する。
* そのため、純粋にJavaScriptのコードを動かしている時間（文字列操作や計算など）の合計が10msを超えなければ、どれだけ長時間の接続やデータベース処理を行っても切断されない。

## 3. 実環境（デプロイ後）のセキュリティと Cloudflare Zero Trust

Cloudflareは世界最高峰のDDoS自動防御（無料プラン含む）が標準装備されているため、デプロイしただけで即座にサーバーが過負荷でダウンするリスクは極めて低いです。さらに保護を固めたい場合は以下が有効です。

### 🛡️ 認証の壁（Cloudflare Access）の挙動
* **デフォルトの挙動**: Zero Trustを契約・有効化しただけでは、**エッジ全体にも個々のWorkerにも勝手にロックはかからない**（誰でもアクセスできる状態が維持される）。
* **ピンポイント設定**: 特定のサブドメイン（例: `://example.com`）や特定のパス（例: `/ws`）だけを指定して門番（Access）を立たせることが可能。

### 🔑 外部プログラム（Goなど）からの接続方法
1. Cloudflareの管理画面で「サービス認証トークン（Service Token）」を発行し、「Client ID」と「Client Secret」を取得する。
2. Goのプログラムで接続する際、HTTPヘッダーにこれらを仕込んでリクエストを送る。

```go
// Go側で認証ヘッダーを付与する実装イメージ
header := make(http.Header)
header.Set("CF-Access-Client-Id", "あなたのClientID")
header.Set("CF-Access-Client-Secret", "あなたのClientSecret")

c, _, err := dialer.Dial("wss://your-worker.pages.dev/ws", header)
```
このヘッダーを持たない不正なアクセスは、Workerが起動して課金される手前（Cloudflareのエッジ）で自動的に弾かれます。
